Your repository should have this structure:

### 3. Test Locally

Before deploying, test the app locally:

```bash

source venv/bin/activate  # On Windows: venv\Scripts\activate

pip install -r requirements.txt

# Run app
streamlit run app/main.py
```

Verify:
- App starts without errors
- All pages load correctly
- Data loads successfully
- Predictions work
- Visualizations render

### 4. Commit and Push

```bash
git add .
git commit -m "Prepare for Streamlit Cloud deployment"
git push origin main
```

## Deployment Steps

### Step 1: Create New App

1. Log in to [share.streamlit.io](https://share.streamlit.io)
2. Click "New app" button
3. You'll see the deployment form

### Step 2: Configure Deployment

Fill in the deployment form:

**Repository**:
- Select your GitHub repository
- Example: `username/engagement-prediction-app`

**Branch**:
- Choose `main` (or your preferred branch)

**Main file path**:
- Enter: `app/main.py`

**App URL** (optional):
- Choose a custom URL subdomain
- Example: `engagement-prediction` → `engagement-prediction.streamlit.app`
- Or leave blank for auto-generated URL

### Step 3: Advanced Settings (Optional)

Click "Advanced settings" to configure:

**Python version**:
- Should auto-detect from `runtime.txt`
- Verify it shows Python 3.10

**Secrets**:
- Add secrets if needed (see Configuration section)
- Use TOML format

### Step 4: Deploy

1. Review all settings
2. Click "Deploy!"
3. Wait for build to complete

**Build Process**:
- Creates virtual environment
- Installs system packages (from `packages.txt`)
- Installs Python packages (from `requirements.txt`)
- Starts Streamlit app

**Typical build time**: 2-5 minutes

### Step 5: Monitor Build

Watch the build logs:
- Check for errors in red
- Verify all packages install successfully
- Wait for "Your app is now deployed!" message

## Configuration

### Secrets Management

If your app needs secrets (API keys, credentials):

1. In Streamlit Cloud, go to your app
2. Click "⋮" menu → "Settings"
3. Click "Secrets" in left sidebar
4. Add secrets in TOML format:

```toml
# Example secrets
[general]
api_key = "your-api-key-here"

[database]
host = "your-db-host"
port = 5432
username = "user"
password = "pass"
```

5. Click "Save"
6. Restart app for secrets to take effect

### Environment Variables

Streamlit Cloud automatically sets:
- `STREAMLIT_SERVER_HEADLESS=true`
- `STREAMLIT_BROWSER_GATHER_USAGE_STATS=false`

To access secrets in code:
```python
import streamlit as st

# Access secrets
api_key = st.secrets["general"]["api_key"]
db_host = st.secrets["database"]["host"]
```

### Custom Configuration

To override default config, modify `.streamlit/config.toml`:

```toml
[server]
maxUploadSize = 200  # MB
maxMessageSize = 200  # MB

[client]
showErrorDetails = false
toolbarMode = "minimal"
```

## Troubleshooting

### Build Failures

**Error: Package installation failed**

Solution:
1. Check package names in `requirements.txt`
2. Verify package versions are compatible
3. Check Streamlit Cloud's supported packages
4. Review build logs for specific error

**Error: Python version mismatch**

Solution:
1. Verify `runtime.txt` contains: `python-3.10`
2. Ensure no conflicting Python version in requirements
3. Check that packages support Python 3.10

**Error: System package missing**

Solution:
1. Add required packages to `packages.txt`
2. Common additions:
```text
libgomp1
build-essential
```

### Runtime Errors

**Error: Module not found**

Solution:
1. Add missing package to `requirements.txt`
2. Ensure package name is correct
3. Reboot app after updating

**Error: File not found**

Solution:
1. Check file paths are relative to app root
2. Verify files are committed to GitHub
3. Check `.gitignore` isn't excluding needed files

**Error: Out of memory**

Solution:
1. Optimize data loading (use chunks)
2. Add caching with `@st.cache_data`
3. Reduce data size
4. Upgrade to Streamlit Cloud Pro

### Performance Issues

**App is slow**

Solutions:
1. Add caching:
```python
@st.cache_data
def load_data():
    return pd.read_csv("data.csv")
```

2. Optimize queries and computations
3. Reduce data size
4. Use pagination for large datasets

**App times out**

Solutions:
1. Reduce initial load time
2. Lazy load data
3. Optimize imports
4. Use background tasks

### Access Issues

**App not accessible**

Solutions:
1. Check app status in dashboard
2. Verify deployment completed
3. Check for errors in logs
4. Try rebooting app

**Secrets not working**

Solutions:
1. Verify TOML syntax in secrets
2. Check secret key names match code
3. Reboot app after adding secrets
4. Don't use quotes around values in TOML

## Best Practices

### Security

1. **Never commit secrets**:
   - Use Streamlit Cloud secrets management
   - Add `.streamlit/secrets.toml` to `.gitignore`

2. **Validate user inputs**:
```python
if uploaded_file:
    if uploaded_file.size > 10_000_000:  # 10MB
        st.error("File too large")
        return
```

3. **Handle errors gracefully**:
```python
try:
    data = load_data()
except Exception as e:
    st.error(f"Error loading data: {e}")
    return
```

### Performance

1. **Cache expensive operations**:
```python
@st.cache_data
def expensive_computation(data):
    # Your computation
    return result
```

2. **Use session state efficiently**:
```python
if 'data' not in st.session_state:
    st.session_state.data = load_data()
```

3. **Optimize data loading**:
```python
# Load only needed columns
df = pd.read_csv("data.csv", usecols=['col1', 'col2'])
```

### User Experience

1. **Show loading states**:
```python
with st.spinner("Loading data..."):
    data = load_data()
```

2. **Provide feedback**:
```python
st.success("Prediction completed!")
st.info("Tip: Try different parameters")
st.warning("Large dataset may take time")
st.error("Invalid input")
```

3. **Handle edge cases**:
```python
if data.empty:
    st.warning("No data available")
    return
```
